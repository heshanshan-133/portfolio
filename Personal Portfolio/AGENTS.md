## 页面布局建议

- **布局模式**: 单栏流式布局（居中最大宽度约束）—— 作品集为阅读型展示页，单栏结构清晰，符合现代个人网站惯例
- **视觉重心**: 作品展示区 —— 用户核心目的是展示项目，作品卡片区域占比最大、视觉最重

- **响应式策略**: 桌面端 2-3 列作品卡片网格 → 平板端 2 列 → 移动端 1 列堆叠

## 功能列表

- **顶部导航栏**:
  - **页面目标**: 提供快速导航入口，增强专业感
  - **功能点**:
    - **滚动高亮**: 滚动到对应区块时，导航项高亮显示当前所在位置
    - **平滑滚动**: 点击导航项平滑滚动到对应锚点

    - **移动端汉堡菜单**: 移动端折叠为汉堡图标，点击展开导航列表

  - **页面目标**: 第一印象展示，快速传达身份与核心信息
  - **功能点**:
    - **姓名与职位展示**: 大字号显示姓名"何姗珊"与职位"游戏开发工程师"
    - **个人简介**: 简短介绍（结合教育背景、GPA、专业排名等亮点）
    - **入场动效**: 页面加载时文字渐入 + 轻微上移动效

- **技术栈展示区**:
  - **页面目标**: 结构化展示技术能力，让招聘方快速掌握技能范围
  - **功能点**:
    - **分类标签展示**: 按"引擎 / 编程语言 / 中间件插件 / 其他"四个分类展示技能标签
    - **悬停动效**: 标签悬停时有缩放/变色微动效

- **作品展示区**:
  - **页面目标**: 核心展示区域，详细呈现项目经验与技术深度
  - **功能点**:
    - **项目卡片网格**: 6个项目卡片以响应式网格布局排列

# UI 设计指南

## 1. 设计推导依据

- **参考意图**: Free Direction —— 附件链接无法访问，按用户文字需求与游戏开发作品集语义自主设计
- **核心情绪 / 应用类型**: 游戏开发工程师求职作品集，需传达专业、创造力与技术深度，重点突出项目展示
- **独特记忆点**: 以游戏引擎界面的精密感为灵感，用细边框 + 微光渐变 + 像素级对齐的卡片网格塑造「开发者工作台」气质，项目卡片悬停时呈现轻微上浮与边缘高光

## 2. Art Direction

- **方向名**: Developer Workbench 深色工作台
- **Design Style**: Minimal Dark + Subtle Glow —— 深色背景降低视觉噪音，让作品卡片成为主角；微光点缀呼应游戏开发的技术感与创造力
- **DNA 参数**: 圆角 `rounded-lg`（中等柔和）/ 阴影 `shadow-sm` + 内发光 / 间距 `gap-6` / `p-6`（标准偏宽）/ 字体方向 无衬线几何 + 等宽技术字 / 装饰手法 细边框、微光渐变、标签式技能芯片
- **应用类型**: Landing / Portfolio —— 单页纵向叙事，作品网格为核心

## 3. Color System

**色彩关系**: 深空灰背景 + 冷青主色 + 低饱和紫调点缀 + 近白文字，整体偏暗调精密感
**配色设计理由**: 深色背景让作品视频占位区与项目卡片自然突出；冷青主色呼应游戏引擎/代码编辑器的技术调性；accent 用低饱和紫，hover 时提供柔和反馈而不刺眼
**主色推导**: 从游戏开发的「引擎界面 + 代码高亮」语义提取冷青色，比纯蓝更有创造力，比霓虹绿更克制专业，适合求职作品集的稳重与个性平衡
**使用比例**: 70% 深中性（bg+card+border）/ 25% 辅助（accent+muted）/ 5% primary；primary 仅用于 CTA、姓名高亮、卡片悬停边框、关键链接

| 角色 | CSS 变量 | Tailwind Class | HSL 值 | 设计说明 |
|---|---|---|---|---|
| bg | `--background` | `bg-background` | hsl(222 20% 8%) | 深空灰，接近夜色的工作台底色 |
| card | `--card` | `bg-card` | hsl(222 18% 12%) | 比背景略亮一层，卡片承载面 |
| text | `--foreground` | `text-foreground` | hsl(210 20% 96%) | 近白正文，高对比可读 |
| textMuted | `--muted-foreground` | `text-muted-foreground` | hsl(215 12% 65%) | 次级说明、元信息 |
| primary | `--primary` | `bg-primary` / `text-primary` | hsl(192 85% 58%) | 冷青主色，引擎界面感 |
| primaryForeground | `--primary-foreground` | `text-primary-foreground` | hsl(222 30% 10%) | 主色上的深色文字 |
| accent | `--accent` | `bg-accent` | hsl(250 25% 18%) | 低饱和紫调，hover/选中浅底 |
| accentForeground | `--accent-foreground` | `text-accent-foreground` | hsl(210 20% 92%) | accent 上的文字 |
| border | `--border` | `border-border` | hsl(222 15% 20%) | 细边界，弱分隔 |

**语义色提示**: 成功 hsl(152 60% 48%) 三态 bg:hsl(152 40% 14%) / border:hsl(152 50% 28%) / text:hsl(152 60% 65%)；警告 hsl(38 80% 55%) 三态 bg:hsl(38 40% 14%) / border:hsl(38 50% 28%) / text:hsl(38 80% 65%)；错误 hsl(0 70% 60%) 三态 bg:hsl(0 30% 14%) / border:hsl(0 40% 28%) / text:hsl(0 70% 70%)；均与主色同低明度基调，饱和度与 primary 对齐 ±15%

## 4. 字体与节奏

- **font-display**: Space Grotesk —— 几何无衬线，带科技感与现代感，适合姓名与职位展示，呼应游戏开发的精密气质
- **font-body**: Noto Sans SC —— 中文正文清晰易读，中性专业，不抢作品风头
- **字号**: H1 text-4xl md:text-5xl；H2 text-2xl md:text-3xl；body text-base；muted text-sm；技能标签 text-xs
- **圆角**: 中等（`rounded-lg`）—— 卡片与按钮统一 8px 圆角，专业中带柔和，避免过于尖锐或过于卡通

## 5. 全局布局契约

- **Reference Layout Use**: 按需求结构推导
- **Page / Section Order**: Hero（导航+姓名+介绍）→ 技术栈 → 作品展示 → 页脚联系方式，单页纵向滚动
- **Standard Content Zone**: `max-w-6xl` + `mx-auto` —— 作品集 Landing 宽度，作品卡片网格 2~3 列舒适
- **Shell / Frame Alignment**: 同宽 —— 导航、内容区、页脚共享同一 max-w 与 padding 节奏
- **Padding & Rhythm**: `px-4 md:px-6 lg:px-8 py-12 md:py-20`，section 间距 `py-16 md:py-24`，保持 8px 倍数
- **Full-bleed Zones**: Hero 背景可全宽延伸，内部文字与导航受 Standard Content Zone 约束
- **Local Narrowing**: 个人介绍正文可收窄至 `max-w-2xl`，保持舒适阅读行长
- **Overflow Strategy**: 技能标签行在移动端使用 `flex-wrap`，不横向滚动
- **Flexibility Boundary**: 允许移动端卡片列数从 2→1、padding 微调；不允许改变主色、圆角、阴影语言、卡片悬停动效逻辑

## 6. 视觉与动效

- **装饰**: 细边框微光 / 渐变描边 / 点状网格背景（极低透明度）
- **阴影/边界**: 轻 —— 卡片默认 `shadow-sm`，悬停时 `shadow-md` + 顶部/边框微光
- **动效**: 精致克制 —— 卡片悬停 `translateY(-4px)` + 边框主色渐变 + 过渡 200ms ease；技能标签 hover 背景色过渡；页面滚动时区块淡入（可选）

## 7. 组件原则

- 作品卡片：默认深灰底 + 细边框，悬停上浮 + 边框冷青微光，内部视频占位区 16:9 灰色底 + 居中播放图标 + "视频占位"文字
- 技能标签：`rounded-full` 胶囊形，深灰底 + 细边框，hover 时背景变 accent
- 按钮：主按钮用 primary 实色填充，次按钮用 outline 边框 + 透明底
- 所有交互元素必须有 `:focus-visible` 状态（2px 主色外描边）

## 8. Image Direction

- **Image Role**: Hero 背景氛围图 + 作品卡片视频占位区替代图（用户后续替换为视频前的临时视觉）
- **Image Art Direction**: Hero 背景——深空灰调下的抽象游戏引擎界面元素（线框、粒子、代码行微光），极低透明度，作为氛围层不抢文字；作品占位图——每个项目可用一张低饱和度风格化封面图替代纯灰框，统一暗调 + 单色光 + 项目主题剪影（如塔防/卡牌/VR 场景轮廓），保持视觉统一
- **Image Prompt Keywords**: dark game engine interface, subtle cyan glow, wireframe particles, minimal tech atmosphere, deep space gray, low opacity background, cinematic lighting, clean composition
- **Image Avoidance**: 避免鲜艳游戏角色海报、通用科技蓝渐变、3D 小人素材、过度复杂的场景图、与项目无关的通用图库照片

## 9. Anti-patterns

- **Split personality**: 深浅色混用或每个卡片风格各异；全站统一深色工作台基调
- **Phantom tokens**: 编造不存在的 CSS 变量；只使用已定义 9 角色 + 语义色
- **Default SaaS drift**: 回到默认蓝白 SaaS 风格；用冷青主色 + 深灰底 + 细边框建立游戏开发专属气质
- **Invisible interaction**: 只做 hover 不做 focus-visible；所有可交互元素必须有键盘可见状态
- **Mono-hue tyranny**: 主色铺满按钮、图标、边框、文字、链接；primary 控制在 5% 以内，其余用 accent / 中性色分层
- **Glow overdose**: 微光效果滥用导致廉价霓虹感；仅卡片悬停边框和 Hero 背景少量使用，保持克制
